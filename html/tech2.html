Parts of an HTML Page

What is Html Page Declaration and Head Document Header Document Body Document Footer
What is HTML

HTML stands for Hyper Text Markup Language. It is a text file that is interpretted by a browser to render web pages and web applications.

What are the sections of a basic HTML document?

  • Page Declaration
  • Head
  • Header
  • Body
  • Footer
Page Declaration and Head

The page declaration lets the browser know what type of document the browser should expect. This is immediately followed by the html tag which will encapsulate the rest of the document. The document declaration has no closing tag. The code below demonstrates the proper use of a declaration tag for html 5 followed by the html tag. Inside the html tag resides the rest of the document's information.

<!DOCTYPE html> <html> </html>

The head is where the metadata for the document belongs. Things you'll find in the head are the title of the page, links, styles, scripts, and other metadata needed to run the page. The head will not be displayed in the browser. It resides between the opening html tag and the opening body tag in the document. Here is a sample of some of the things you might find in the head.

<head> <meta charset="utf-8" /> <title>Page Title</title> <style> </style> </head>
Document Header

A document header is usually at the top of the page. It is where introductory content and/or links are placed. You can have more than one per document, but there is usually a main one at the top.

The code below demonstrates a basic header that has no links, just a big title.

<header> <h1> Hello Everyone! </h1> </header>
Document Body

The body contains the content of the page. It has all the pictures, the words, and the hyperlinks of the document. Everything that is not the page declaration,the html tag, or the head should reside within the body. The body resides within the html tag and below the head tag.

Here is an example of the body tag properly placed.

<html> <head> </head> <body> </body> </html>
Document Footer

Not every document has a footer, but they can be useful in certain situations. Like headers, you can have more than one footer per page and they can contain links. The purpose of footers is usually to reference information, tell you about who wrote it or who owns it, or just give you a link back to the top so you don't have to scroll. Contact information and site maps are usually located here as well. A footer will go at the bottom of an element and the information is usually specific to that element.

In this example, the footer is at the bottom of the body tag, so it would probably contain information about the entirity of the document.

<html> <head> </head> <body> <header> </header> <footer> </footer> </body> </html>
